Setup a Ruby on Rails (RoR) environment on OSX

1. Basic Environment

  Install Rails on your system (didn’t work?)
  Install HomeBrew
  Install MySQL (follow brew's instructions at the end)
  Initial setup
  Launch at startup
  Create a new RoR app
 
  Create your Database
  Start Up your Web App

 

Notes:

  1. You don’t need to install Homebrew and MySQL in order to build a RoR app, having a database is an option and you can choose to use alternatives like SQLite, PostGgeSQL or MongoDB, etc;
  2. In case you need to work on multiple RoR projects and each of them requires a different set of ruby version and gems, take a look on How to set Multiple RoR Projects.

2. Explanation

To develop in PhP you most certainly would be using an AMP (Apache, MySQL, PhP and the mod_php for Apache) like MAMP. Sadly there is no MAMP equivalent for RoR, but fortunately OSX comes with ruby and rubygems already installed. Try “ruby -v” and “gem -v” on your terminal to verify.

With ruby installed, your system will be able to interpret ruby language and with rubygems, you’ll be able to easily install gems (or plugins if you will), like Rails.

I guess you can picture rubygems like the apt-get or Homebrew, for ruby.

By having Ruby installed, you'll also have WEBrick Web Server, which comes with Ruby. Its a Ruby Web Server that in a PhP scenario is the equivalent of installing Apache with mod_php.

So, you already have Ruby <-> PhP, WEBrick <-> Apache + mod_php, install MySQL and you've got yourself the equivalent of a PhP AMP, so in reality you only need to install Rails and a database (should you need one) in order to build and run a RoR app.

Question: If I already have, what it is considered, an PhP AMP equivalent, right out-of-the-box, why do I need to install Rails?
Answer: Because you're creating an Ruby on Rails app, and not just a Ruby app.

 

Advice: To install MySQL I strongly advise you to install Homebrew it will help you to easily install MySQL and some cool stuff like Redis, ImageMagick and others.

Note: You'll need GCC in order to compile Rails, so in case "gcc -v" fails on your terminal, then you need to download "Command Line Tools for XCode".

3. Connecting the dots

  1. With ruby installed, your system will be able to interpret ruby language;
  2. With rubygems, you’ll be able to install gems like Rails;
  3. By installing Rails (sudo gem install rails), rubygems will install other gems that Rails depends on and one of them is Bundler.
    By now you can create a new RoR App by running the command “rails new my-app-name -d mysql”;
  4. With Bundler, you’ll be able to easily install/uninstall all the gems that you add/remove to a file inside your app’s folder called “Gemfile”, by running the command “bundle” on your app folder;
  5. When Rails finishes creating a new app, it will run the command “bundle” that will try to install any gems listed on the "Gemfile" and due to the "-d mysql" option, MySQL Gem will be one of them.
    Note: You’ll need to have MySQL installed first in order for the MySQL Gem to work;
  6. Running the command "rake db:create", you will be instructing a build tool called Rake (that comes with Ruby) to connect to your MySQL Server and create a clean database, according to what it is configured on your "config/database.yml" YAML file;
  7. Now if you run the command “rails s” on your app’s folder, Rails will launch WEBrick (its a Ruby Web Server, the equivalent of Apache + mod_php) and you’ll be able “see” your app in the browser: http://localhost:3000, this is the equivalent off starting the MAMP app when you want to “see” your PhP app: http://localhost:8888.
