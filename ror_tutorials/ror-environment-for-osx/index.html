Setup a Ruby on Rails (RoR) environment on OSX

Published on 23-11-2012

1. Basic Environment

Suppose you want to develop in PhP. You’d have to have Apache, probably MySQL, PhP and the mod_php for Apache, that combination is called an AMP and the most common AMP installer for OSX: MAMP installs just that.

So, to develop in RoR you’ll need the same things. There is no MAMP equivalent for RoR, but fortunately OSX comes with ruby and rubygems already installed. Try “ruby -v” and “gem -v” on your terminal to verify.

With ruby installed, your system will be able to interpret ruby language and with rubygems, you’ll be able to easily install gems (or plugins if you will), like Rails and Mongrel a ruby web server that in a PhP scenario is the equivalent of installing Apache with mod_php.

I guess you can picture rubygems like the apt-get or Homebrew, for ruby.

So basically you only need to install Rails and GCC in order to compile Rails, so in case you don't have XCode or a GCC compiler installed, I advise you to download "osx-gcc-installer".

Optionally, you might need a Database and for that I strongly advise you to install Homebrew it will help you to easily install MySQL and some cool stuff like Redis, ImageMagick and others.

3. Resume

I guess all this new stuff can be a bit overwhelming at first, so I’ll resume it down to a 5 story steps so you can connect all the dots:

  1. With ruby installed, your system will be able to interpret ruby language;
  2. With rubygems, you’ll be able to install gems like Rails;
  3. By installing Rails, rubygems will install other gems that Rails depends on and one of them is Bundler.
    By now you can create new RoR app by running the command “rails new my-app-name -d MySQL”;
  4. With Bundler, you’ll be able to easily install/uninstall all the gems that you add/remove to a file inside your app’s folder called “Gemfile”, and one off these gems will be a Ruby Web Server (for instance Mongrel).
    By running the command “bundle” on your app folder, Bundler will use rubygems to install Mongrel (if you choose to use it) and newly listed gems.
    Note: You’ll need to have MySQL installed first in order for bundle to work since it will install the MySQL Gem (because it is listed on your “Gemfile” due to the “rails new my-app-name -d MySQL” option);
  5. Now if you run the command “rails s” on your app’s folder, Rails will launch Mongrel (or any other Ruby Web Server) and you’ll be able “see” your app in the browser (http://localhost:3000), this is the equivalent off starting the MAMP app when you want to “see” your PhP app (http://localhost:8888);
