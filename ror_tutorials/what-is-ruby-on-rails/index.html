What is Ruby on Rails

1. Introduction

Being a web programmer, chances are, you’ve used PhP before and hopefully with a WebFramework like CodeIgniter, so I’ll make a simple analogy between PhP and Ruby on Rails (RoR) development so you can have a point of reference.

So in a nutshell Ruby on Rails is you using Ruby language (PhP) with Rails (CodeIgniter) a real web Framework that prevents you from writing redundant code, enforces some best-practices habits and makes you enjoy writing code, again. This will require you to learn the Ruby syntax and get familiar with Rails (see section 2).

Using the most common scenario where you are trying to create a simple PhP app, you’ll probably be buying a remote server that typically comes with cpanel and some pre-installled cool stuff like apache, php, etc.

What would you need to have in order to deploy the same app, made in RoR?

PhP

RoR
1. WebServer Apache Apache
2. Programming Language PhP Ruby
3. Bridge between the WebServer and the Programming Language Interpreter mod_php Passenger
4. DataBase MySQL MySQL
5. Web Framework CodeIgniter Rails

In the RoR case, you’ll also be needing one more thing: Ruby Gems so you can easily install/uninstall some cool Gems (or Plugins if you will), like Rails that will make you think that your app runs on magic dust.

2. First you need to learn:

  1. Ruby, like you learn PhP (see Ruby in Twenty Minutes);
  2. Rails, like you’d have CodeIgniter should you use it (see Rails for Zombies);
  3. Set up a RoR environment on your machine (see OSX RoR environment);
  4. Create your first app, doesn’t matter what it is has long has you get your hands dirty (see Youtube Video);
  5. and finally learn how to deploy your app to a production server (see Deploy RoR app to AWS EC2).
