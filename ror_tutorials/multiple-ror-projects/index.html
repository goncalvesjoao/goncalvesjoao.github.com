Working on Multiple RoR Projects

Published on 23-11-2012

Say you have two projects: project1 that requires ruby v1.9.2 and rails v3.2.1 and project2 that requires ruby v1.8.7 and rails 2.3.5.

Without a Ruby Version Manager like rvm, in case you're working on project1 and decide to work on project2, you'd have to replace you system's ruby (v1.9.2 to v1.8.7) and gems (rails v3.2.1 to v2.3.5) that project2 needs, making it a pain in the ass if you want to start working on project1 again.

With rvm you can install different ruby versions on your machine: "rvm install 1.9.2", "rvm install 1.8.7" and switch between them "rvm use 1.9.2", "rvm use 1.8.7", without being stuck to your system's ruby version.

The same goes for gems. Suppose project1 needs rails v3.2.1, paperclip v3.0 and other gems, project2 needs rails v2.3.5, paperclip v2.7 and some others, by creating the gemsets "rvm gemset create project1_gemset" and "rvm gemset create project2_gemset" you can install the gems that each project needs on the correspondent gemset and easily switch between them "rvm gemset use project1_gemset" and "rvm gemset use project2_gemset".

Even cooler is the option to create a ".rvmrc" file on an app's root folder, with (rvm 1.9.2@project1_gemset) the ruby version and gemset that you want to load for that app when you ("cd project1/") enter that app's folder.

Steps to create a new project using ruby version “ree 1.8.7” and rails “3.0.5”:

local machine

Install rvm

Reopen your terminal or run the following command

Install Ruby (can by any version you want) in the rvm context

DIDN’T WORK? You need to have GCC installed, get “osx-gcc-installer”, install it and try installing Rails again.
Note: Should you see a warning message, ignore it and hit 'q' to continue.

Start using the newly installed Ruby version

Create a gemset “my-app-gemset”

Switch to (or start using) that “my-app-gemset” context

Instal the Rails 3.0.5 version (again, can be any version you want) in that “my-app-gemset” context

Create your new app

Move to “my-app” folder

Create a file called “.rvmrc”

DIDN’T WORK?“open -a TextEdit .rvmrc” if you don’t have TextMate.

Paste the following: “rvm ruby-version@your-gemset-name”

 

Now every time you move to your app’s folder ("cd my-app/") rvm will read the “.rvmrc” file and will load the selected ruby version “ree” and the set of gems that where installed in the “my-app-gemset” context.

Note: If you’re having trouble with some missing gems, try running the “bundle install” command.
