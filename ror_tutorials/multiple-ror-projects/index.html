Working on Multiple RoR Projects

1. Creating an app using: Ruby 1.9.3 + Rails 3.2.8

  Install rvm
  Reopen your terminal or run the following command
  Install the Ruby version you need in the rvm context (didn’t work?)
  Start using the newly installed Ruby version
  Create a gemset “your-gemset-name”
  Start using that “your-gemset-name” gemset
  Instal the Rails version you need in that “your-gemset-name” context
 
  Create a new RoR app
 
  Create a file called “.rvmrc” (don't have TextMate?)
  Add the following: “rvm ruby-version@your-gemset-name” and save it

 

Notes:

  1. Now every time you move to project1’s folder ("cd project1"), rvm will read the “.rvmrc” file and will load the selected ruby version “1.9.3” and the set of gems that where installed in the “project1_gemset” context.
  2. If you’re having trouble with some missing gems, try running the “bundle install” command.

2. Explanation

Say you have two projects: project1 that requires ruby v1.9.3 and rails v3.2.8 and project2 that requires ruby v1.8.7 and rails v3.0.5.

Without a Ruby Version Manager like rvm, in case you're working on project1 and decide to work on project2, you'd have to replace your system's Ruby (v1.9.3 to v1.8.7) and Gems (rails v3.2.8 to v3.0.5) that project2 needs, making it a pain in the ass if you want to start working on project1 again.

With rvm you can install different ruby versions on your machine: "rvm install 1.9.3", "rvm install 1.8.7" and switch between them "rvm use 1.9.3", "rvm use 1.8.7", without being stuck to your system's ruby version.

The same goes for gems. By creating the gemsets "rvm gemset create project1_gemset" and "rvm gemset create project2_gemset" you can install the gems that each project needs on the correspondent gemset and easily switch between them "rvm gemset use project1_gemset" and "rvm gemset use project2_gemset".

Even cooler is the option to create a ".rvmrc" file inside your app's root folder, that will instruct rvm to load a certain Ruby version and Gems installed on a specific gemset, when you "cd project1/".

3. Working on an unknown project

You wanna clone and work on an unknown RoR project, like: git://github.com/railscasts/106-time-zones-revised.git.
The only thing you know is that it requires Ruby 1.9.2 or higher.

 

 
  Clone the project
 
  Install the correct Ruby version
  Load the correct Ruby version (install it first, if you haven't already)
  Create a gemset for this project
  Load it
  Install all the necessary Gems on the project's gemset
  Create and initiate the project's database
  Start up the app

 

Advice: While you're at it, create a ".rvmrc" file with "rvm 1.9.3@youhaiku-after" in it, so you don't have to constantly remember and manually load the Ruby and Gemset for this particular project.
